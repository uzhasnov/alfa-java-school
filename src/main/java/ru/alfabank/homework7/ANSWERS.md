1. Почему heroesCreated должно быть статическим? 
Переменная отвечает за общее количество созданных героев и относится к классу, а не к конкретному экземпляру класса.

2. Почему name, health и level не должны быть статическими?
Потому что эти поля описывают состояние конкретного героя и принадлежат экземпляру класса.

3. Почему printHeroesCreated() можно вызвать без создания объекта?
Потому что метод помечен ключевым словом static и относится к классу.

4. Можно ли из статического метода напрямую обратиться к нестатическому полю name? Почему?
Нельзя. Статический контекст инициализируется раньше, чем происходит вызов конструкторов и создаются объекты. Поэтому
такие обращения запрещены.

5. Почему проект перестанет компилироваться если пометить класс Hero модификатором final?
Ключевое слово final на уровне класса запрещает от такого класса наследоваться. Поэтому классы Knight, Archer и Mage
не смогут унаследоваться от Hero.

6. Почему переменные имеют тип Hero, но при этом вызываются версии метода attack() из конкретных наследников?
Тип переменной, в которую сохраняется ссылка на объект определяет скоуп возможностей для объекта, а тип самого объекта 
определяет реализацию поведения. Поскольку, метод attack() изначально определён в классе Hero - наследникам будет 
доступен вызов этого метода из переменной типа Hero (благодаря связи IS A и полиморфизму). И, поскольку, метод attack() 
переопределён в классах-наследниках - реализация(тело метода) будет определяться типом объекта, т.е. будет браться у 
одного из классов-наследников.

7. Почему переменная knight имеет модификатор final, но её поля могут быть изменены?
Потому что это стандартное поведение для final переменных ссылочного типа. В переменную ссылочного типа сохраняется 
ссылка на объект и и эты ссылка неизменна после создания объекта, т.е. в такую переменную нельза сохранить другой объект. 
При этом, состояние исходного объекта изменять не запрещается.
